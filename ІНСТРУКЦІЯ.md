# WMS — Інструкція з підключення до Supabase

## Що вже готово

- ✅ Логіка всіх операцій, інвентаризація, звіти, QR
- ✅ SQL-файли для створення бази даних
- ✅ Код синхронізації між пристроями

## Що потрібно зробити (5 кроків, ~30 хвилин)

---

### КРОК 1: Створити таблиці в Supabase (5 хвилин)

1. Відкрийте https://supabase.com/dashboard
2. Оберіть ваш проєкт **wms-100life**
3. Зліва натисніть **SQL Editor** (іконка з кодом)
4. Натисніть **+ New query**
5. Скопіюйте ВЕСЬ вміст файлу `supabase/migrations/001_create_tables.sql`
6. Вставте в поле запиту
7. Натисніть зелену кнопку **Run** (або Ctrl+Enter)
8. Повинно написати "Success"

Повторіть те саме для файлів:
- `002_rls.sql`
- `003_realtime.sql`  
- `004_seed.sql`

**Порядок важливий!** Спочатку 001, потім 002, потім 003, потім 004.

---

### КРОК 2: Створити першого користувача (3 хвилини)

1. В Supabase Dashboard зліва натисніть **Authentication**
2. Натисніть **Users** → **Add user** → **Create new user**
3. Введіть:
   - Email: `admin@100life-dniproregion.org.ua` (або ваш email)
   - Password: придумайте надійний пароль
   - Увімкніть **Auto Confirm User**
4. Натисніть **Create user**
5. Тепер відкрийте **SQL Editor** знову
6. Скопіюйте вміст `005_create_admin.sql`
7. Якщо ви використали інший email — замініть його в запиті
8. Натисніть **Run**

---

### КРОК 3: Знайти ваш Supabase ключ (2 хвилини)

1. В Supabase Dashboard зліва натисніть **Settings** (шестерня)
2. Натисніть **API**
3. Знайдіть секцію **Project API keys**
4. Скопіюйте значення **anon public** (це довгий рядок типу `eyJhbGciOi...`)
5. Збережіть його — він потрібен на наступному кроці

⚠️ НЕ діліться цим ключем публічно, але anon key — це не секрет, його можна використовувати у фронтенд-коді.

---

### КРОК 4: Налаштувати Vercel (3 хвилини)

1. Відкрийте https://vercel.com/dashboard
2. Натисніть на ваш проєкт **wms**
3. Натисніть **Settings** (вгорі)
4. Зліва натисніть **Environment Variables**
5. Додайте дві змінні:

| Key (назва)              | Value (значення)                                    |
|--------------------------|-----------------------------------------------------|
| VITE_SUPABASE_URL        | https://hnpyzghqsjexhnfozcse.supabase.co            |
| VITE_SUPABASE_ANON_KEY   | (вставте anon key з кроку 3)                        |

6. Натисніть **Save** для кожної

---

### КРОК 5: Завантажити код на GitHub (10 хвилин)

Це єдиний крок де потрібна допомога розробника АБО ви можете зробити це самі через GitHub web-інтерфейс.

**Варіант А: Через GitHub (без програмування)**
1. Відкрийте https://github.com/100life-wms/wms
2. Для кожного файлу: натисніть **Add file** → **Upload files** або **Create new file**
3. Завантажте файли з папки проєкту

**Варіант Б: Попросити розробника**
Передайте цей ZIP-архів розробнику і скажіть:
"Потрібно замінити файли в репозиторії 100life-wms/wms цими файлами і зробити push в main"

Після push Vercel автоматично оновить сайт wms-ten-lake.vercel.app

---

## Після налаштування

Відкрийте https://wms-ten-lake.vercel.app і увійдіть з email/паролем який ви створили в Кроці 2.

## Якщо щось не працює

- **"Invalid login"** — перевірте що email і пароль правильні, що користувач створений з Auto Confirm
- **Дані не зберігаються** — перевірте що env vars в Vercel правильні, натисніть Redeploy
- **Порожній екран** — відкрийте консоль браузера (F12), скопіюйте помилку і покажіть мені
